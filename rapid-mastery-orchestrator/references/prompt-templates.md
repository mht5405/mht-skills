# Prompt模板库

所有模板均需根据用户的具体目标领域进行适配。

---

## 1. 知识图谱生成

```
给我[产品经理]领域的完整知识图谱，包括：
- 核心概念与术语表
- 关键流程与框架（如JTBD、RICE、OKR等）
- 2026年最新趋势（AI与领域结合、数据驱动等）
- 高阶心智模型
- 推荐免费/经典资源（书籍章节、文章、案例）

结构要求：先给我一个宏观分类，再逐层展开。
```

## 2. 基线测试

```
基于以上知识图谱，出一套10题基线测试题，覆盖[产品经理]领域必须掌握的核心内容。
题型包括：概念理解、场景应用、案例分析。
给完题目后，等我提交答案再批改。
```

## 3. 结构化讲解

```
用结构化方式解释[RICE优先级框架]，像教零基础但聪明的人：
1. 定义：一句话说清楚是什么
2. 为什么重要：解决什么核心问题
3. 如何应用：分步骤讲解，配真实案例
4. 常见坑：新手最容易犯的错误
5. 进阶思考：什么情况下这个框架不适用？
```

## 4. Feynman检查（用户向LLM输出）

```
我现在用最简单的话解释[概念名称]，你扮演12岁小学生，指出我解释中：
1. 不准确的地方
2. 遗漏的关键点
3. 可以用更简单的比喻替换的地方
4. 追问3个让我必须深入理解才能回答的问题

我的解释：
[用户写下自己的解释]
```

## 5. 严格批改

```
作为10年经验的资深[产品经理]，请严格、专业、无情地批改这份[PRD]：

[用户粘贴自己的输出]

批改要求：
1. 逐段指出问题，说明为什么是问题
2. 给出具体的改进版本
3. 按1-10分打分（10分=可直接提交给团队）
4. 给出3个最需要优先改进的点
5. 解释你的评分逻辑
```

## 6. 真实场景练习生成

```
给我5个真实场景练习题，要求我产出[PRD/用户故事/Roadmap]。
题目难度递增（从简单到复杂），每个场景包含：
- 背景信息
- 利益相关者
- 约束条件
- 目标产出物描述
我先做完第1题你再出第2题，每次都要严格批改。
```

## 7. 资源推荐

```
推荐2026年最实用的[产品经理]免费资源和经典书关键章节总结：
- 5个免费资源（博客、播客、newsletter）
- 3本经典书，每本只给我最重要的2-3个核心章节摘要
- 每个资源说明：为什么有用 + 什么时候看
```

## 8. Anki卡片生成

```
基于我今天学的[RICE框架/用户研究/PRD写作]，生成10张高质量Anki卡片：
- 5张概念卡片（定义/公式/模型）
- 3张应用场景卡片（给案例让我判断用哪个框架）
- 2张反例卡片（常见错误或陷阱）

格式：
Q: [问题]
A: [答案]
标签: #[模块名]
```

## 9. 角色扮演：CEO对话

```
现在是角色扮演。
你是一家SaaS创业公司的CEO。
我向你展示我的产品方案[用户描述方案]。
请：
1. 以CEO身份提出3个尖锐的商业问题
2. 指出方案中商业逻辑的漏洞
3. 追问数据和ROI依据
4. 最后给出你的判断：这个方案值不值得继续投入？

注意：你的问题必须尖锐、真实，不要客气。
```

## 10. 角色扮演：Mock面试

```
你是一家一线互联网公司的[产品经理]面试官。
请按以下流程进行Mock面试：
1. 行为问题（2题）—— 关于过往项目经验和决策
2. 案例分析（1题）—— 给出一个产品场景让我分析
3. 专业问题（2题）—— 考察领域核心概念理解

每道题我回答后，你：
- 给出评分（1-10）
- 指出回答的亮点和不足
- 给出一个更优的回答示范
```

## 11. 多模型陪审团指令

```
请对以下[PRD]从三个角度同时批改：
1. 作为产品VP（战略完整性）
2. 作为资深PM（执行细节）
3. 作为工程师（技术可行性）

每个角度给出独立评分和改进建议。
最后给出综合评分和优先级最高的3个改进点。

[用户粘贴输出]
```

## 12. 元认知日志 Prompt

```
今天的学习日志：
1. 今天学到了什么（用一句话概括最重要的收获）？
2. 哪里卡住了？（具体到哪个概念/技能/步骤）
3. 为什么卡住？（知识不够/方法不对/注意力不集中）
4. 明天我该如何改进？

请分析我的回答，识别潜在风险（拖延/浅尝辄止/虚假自信），给出针对性的改进建议。
```

## 13. 成果集自动生成 Prompt

```
基于我这7天的所有里程碑输出，帮我生成一份精美的「7天[产品经理]Mastery作品集」。

作品集内容：
1. 标题页（领域 + 时间 + 一句话总结）
2. 学习路径概览（7天经历的关键里程碑）
3. 每个里程碑的核心产出（每个输出配简短说明）
4. 反馈与迭代记录（LLM批改和改进过程）
5. 自我复盘（学到了什么、最大的突破、还待提升的地方）
6. 下一步学习方向

格式要求：专业、简洁、可直接对外展示（HR/面试官）。
```

## 14. 自适应反拖延指令

```
我最近两天的输出质量持续下降。请分析可能原因，并启动「极简模式」：
1. 今天的输入时间压缩到30分钟
2. 只做1个核心输出
3. Feynman输出只写3句话
4. 给我一个"最简可行方案"来完成今天的里程碑
请直接给我今天的极简任务清单。
```

---

## 模板使用规则

- **Day 1**: 模板1（知识图谱）+ 模板2（基线测试）+ 模板12（元认知日志）
- **Day 2-5**: 模板3-8（讲解+Feynman+批改+练习+Anki）
- **Day 6**: 模板13（成果集）
- **Day 7**: 模板9-10（角色扮演+Mock面试）
- **持续**: 模板11（陪审团）+ 模板12（元认知）+ 模板14（自适应反拖延）
