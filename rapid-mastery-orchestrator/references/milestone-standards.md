# 里程碑关卡标准

每个里程碑有明确的"通过标准"。用户必须达到标准才能进入下一阶段。

---

## M0 — 准备（Day 0）

**通过条件**：以下全部完成
- [ ] 建立Notion/Obsidian Workspace（含5个分区：知识图谱、每日笔记、项目成果、Anki卡片、反馈记录）
- [ ] 安装Anki并了解基本用法
- [ ] 确认主要使用的LLM工具（Claude/GPT/Grok）
- [ ] 发布了公开承诺（朋友圈/X/Notion公开链接）
- [ ] 设定了失败惩罚（具体、可执行）
- [ ] 找好了责任伙伴并确认监督机制
- [ ] 明确了7天6个里程碑的具体交付物

**未通过处理**：补全缺失项后进入Day 1

---

## M1 — 领域地图与基线评估（Day 1）

**通过条件**：LLM评分≥85分，且以下全部完成
- [ ] 生成完整知识图谱Mindmap（至少覆盖核心概念、关键流程、常用框架、最新趋势）
- [ ] 完成10题基线测试，LLM批改并识别出3个以上弱点
- [ ] 输出"当前水平 vs 胜任差距"文档
- [ ] 完成了Feynman输出（至少对3个核心概念进行解释-反馈迭代）
- [ ] 生成了10张Anki卡片
- [ ] 完成了元认知日志

**评分标准**：
- 知识图谱覆盖率（30分）：是否覆盖80%的核心概念
- 基线测试正确率（30分）：≥60%正确
- Feynman输出质量（20分）：LLM反馈显示理解基本正确
- 差距分析深度（20分）：能清晰识别自己的3个以上弱点

**低于85分**：补做Feynman输出，重新迭代，LLM认为"理解正确"后再提交

---

## M2 — 用户研究与问题定义（Day 2）

**通过条件**：LLM评分≥85分，且以下全部完成
- [ ] 完成领域相关的用户研究方法论学习（Feynman输出）
- [ ] 产出了至少2个Persona（含用户画像、痛点、目标、行为模式）
- [ ] 产出了用户访谈脚本（至少10个问题）
- [ ] 完成了项目应用（将Persona和脚本应用到自己的项目idea中）
- [ ] LLM严格批改后，至少迭代了1版
- [ ] 生成了10张Anki卡片
- [ ] 完成了元认知日志

**评分标准**：
- Persona深度（30分）：是否基于真实用户洞察而非刻板印象
- 访谈脚本质量（30分）：问题是否开放、无引导、能挖掘深层需求
- 项目应用（20分）：是否真正应用到自己的项目场景
- 迭代改进（20分）：根据LLM反馈进行了实质性修改

**低于85分**：重新梳理Persona，找LLM做"用户研究深度对话"模拟

## M3 — 战略与市场分析（Day 3）

**通过条件**：LLM评分≥85分，且以下全部完成
- [ ] 完成核心战略框架的学习（如SWOT、波特五力、PMF分析等，Feynman输出）
- [ ] 产出了市场/竞争分析文档
- [ ] 完成了产品战略定位文档
- [x] **真实世界验证**：将前两天的Persona或访谈脚本发送到至少3个真实目标用户群（小红书/微信群/Reddit等），并**拿到反馈截图或记录**
- [ ] LLM严格批改后迭代
- [ ] 生成了10张Anki卡片
- [ ] 完成了元认知日志

**评分标准**：
- 战略分析深度（30分）：框架运用是否合理，洞察是否深入
- 真实反馈证据（30分）：**必须提供外部反馈的截图/记录**
- 战略与用户研究的连贯性（20分）：Day 2的用户洞察是否在Day 3的战略中体现
- 迭代改进（20分）

**低于85分**：重点强化真实反馈部分，如果外部反馈不够则延长时间继续收集

## M4 — 优先级排序与需求管理（Day 4）

**通过条件**：LLM评分≥85分，且以下全部完成
- [ ] 完成优先级框架学习（RICE、MoSCoW、Kano等，Feynman输出）
- [ ] 产出了完整的PRD（至少包含：背景、目标、用户故事、验收标准、优先级、范围）
- [ ] 产出了需求优先级排序文档（含决策逻辑和理由）
- [ ] 完成了项目应用（将PRD应用到自己的项目idea中）
- [ ] LLM严格批改后至少迭代了1版
- [ ] 生成了10张Anki卡片
- [ ] 完成了元认知日志

**评分标准**：
- PRD完整性（35分）：是否包含所有必要模块
- 优先级决策逻辑（30分）：决策过程是否清晰、有依据
- 用户故事质量（20分）：是否符合INVEST原则

**低于85分**：重点改进PRD，LLM给出每段的具体修改建议

- [ ] 整合前5天的碎片输出，形成完整的端到端案例

- [ ] **真实世界验证**：完整作品集发给至少1位外部PM/行业人士获取反馈
- [ ] LLM严格批改后迭代

- [ ] 完成了元认知日志

**评分标准**（本关卡评分提高到88分及格）：

- 商业判断力（25分）：方案是否考虑了商业可行性
- 外部反馈证据（25分）：**必须提供外部行业人士的反馈**
- 原型质量（15分）：是否能清晰传达产品概念

**低于88分**：根据LLM和外部反馈逐项改进，直到通过

## 自适应调整规则

| 情况 | 调整动作 |
|------|---------|
| 连续2天输出质量下降（LLM评分<70） | 强制启动"极简模式"：输入时间压缩至30min，只做1个核心输出 |
| 里程碑3次未通过 | 回到上一个里程碑重新巩固，并提供额外辅导 |
| 用户情绪明显低落/拖延 | 启动元认知干预：分析原因、降低当天期望、给1个简单任务重建信心 |
| 用户时间不足（每天<4h） | 将7天扩展为14天，每天只做核心任务 |
| 用户发现框架不适用 | 根据用户反馈调整里程碑内容，但保留核心循环 |
